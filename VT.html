Introduction to Database System

Introduction

What is database architecture?
The term "database architecture" refers to the structural design and methodology of a database system, which forms the core of a Database Management System (DBMS). This architecture dictates how data is stored, organized, and retrieved, playing a crucial role in the efficiency and effectiveness of data management.

Functional components of a database system include:

  • Storage Manager
  • Query Processor

Database System Architecture

1. Disk Storage

  • All data is stored in disk storage.
  • Components of disk storage:
    • Data: Stores the database itself.
    • Data Directory: Contains metadata such as table schemas.
    • Indices: Provide fast access to specific data values.
    • Statistical Data: Stores statistics about the stored data.

2. Storage Manager

The Storage Manager acts as an interface between low-level data storage, application programs, and system queries.

  • Interacts with the file manager.
  • Translates DML statements into file system commands.
  • Stores, retrieves, and updates data.

3. Query Processor

The Query Processor simplifies and facilitates access to data.

  • DDL Interpreter: Interprets and records DDL statements.
  • DML Compiler: Translates DML queries into low-level instructions and optimizes queries.
  • Query Evaluation Engine: Executes low-level instructions generated by the DML Compiler.

Database Architecture Types

Two-Tier Architecture

  • Based on Client-Server architecture.
  • Direct communication between client and server.
  • Common in small-scale applications.

Three-Tier Architecture

  • Separates application into three layers:
    • Client Layer: UI for user interaction.
    • Business Layer: Contains business logic and acts as an intermediary.
    • Data Layer: Manages database storage and access.
